# 🌐 What is GitHub?

**GitHub** is a cloud-based platform that hosts Git repositories online.

It allows developers to:

- Store code online
- Collaborate with others
- Track project history
- Showcase projects as a portfolio

> **Think of GitHub as Google Drive for your code.**

---

# 📦 What is a Repository?

A **Repository (Repo)** is a place where your project's files, folders, and version history are stored.

Every project on GitHub is stored inside a repository.

### 📌 Repository Contains

- Project files
- Folders
- README
- Commit history
- Version history

---

# 💻 Create a New Repository

Follow these simple steps:

1. Sign in to your GitHub account.
2. Click the **➕ New** button.
3. Enter a **Repository Name**.
4. (Optional) Add a **Description**.
5. Choose **Public** or **Private**.
6. (Optional) Check **Add a README file**.
7. Click **Create Repository**.

> 📍 **Image:** `Images/create-github-repository.png`

---

# 📝 Repository Information

When creating a repository, you'll usually provide:

| Field | Purpose |
|--------|----------|
| Repository Name | Name of your project |
| Description | Short summary (optional) |
| Visibility | Public or Private |
| README | Introduces the project |
| .gitignore | Ignores unnecessary files |
| License | Defines how others can use your project |

---

# ✅ Key Points

- GitHub stores Git repositories online.
- A repository contains your project and its history.
- Every project should have a meaningful repository name.
- Adding a README makes your project easier to understand.

# 🌍 Public vs Private Repository

When creating a repository, GitHub lets you choose its visibility.

| Public Repository | Private Repository |
|-------------------|--------------------|
| Anyone can view the repository | Only you and invited collaborators can access it |
| Great for portfolios and open-source projects | Best for personal or confidential projects |
| Can be discovered by others | Hidden from the public |

> 💡 **Tip:** If you're learning web development and building projects, creating **Public** repositories is a great way to build your portfolio.

---

# 🖥️ Explore the GitHub Repository Interface

After creating a repository, you'll see several tabs.

| Tab | Purpose |
|-----|---------|
| **Code** | View project files and source code |
| **Issues** | Track bugs, tasks, and feature requests |
| **Pull Requests** | Review and merge code changes |
| **Actions** | Automate workflows using GitHub Actions |
| **Projects** | Organize tasks and project planning |
| **Wiki** | Create project documentation |
| **Settings** | Manage repository settings |

> 📍 **Image:** `Images/github-interface.png`

# 📂 Common Repository Files

Some files are commonly found in most GitHub repositories.

### 📄 README.md

- Introduces the project
- Explains how to use it
- Usually the first file visitors read

---

### 🚫 .gitignore

Lists files and folders that Git should **ignore**.

Examples:

- Temporary files
- Cache files
- Build folders
- Environment files

---

### ⚖️ LICENSE

Defines how others can use, modify, or share your project.

---

# 💡 Best Practices

- ✅ Use a clear repository name.
- ✅ Add a short description.
- ✅ Include a README file.
- ✅ Keep repositories organized.
- ✅ Make learning projects **Public** to showcase your work.

---

# ✅ Key Points

- Public repositories are visible to everyone.
- Private repositories are accessible only to authorized users.
- The **Code** tab is where you'll spend most of your time.
- A good README helps others understand your project.
