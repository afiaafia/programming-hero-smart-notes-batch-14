# рҹҺЁ Design Tools: Figma, Penpot & UI Design Workflow

Before writing any code, professional frontend developers usually receive a **UI/UX Design** from a designer.

Instead of guessing how the website should look, they follow the design carefully and convert it into code.

This process is called **Design-to-Code Workflow**.

---

# рҹҺЁ What is Figma?

**Figma** is one of the world's most popular **UI/UX Design** tools.

Designers use Figma to create:

- рҹ“ұ Mobile App Designs
- рҹ’» Website Designs
- рҹҺЁ User Interfaces (UI)
- рҹ§© Components
- рҹ–јпёҸ Icons & Illustrations
- рҹ“җ Design Systems

> рҹ’Ў Figma is cloud-based, so you can open your designs directly in your web browser without installing heavy software.

---

# вңЁ Why Developers Use Figma

Frontend developers don't use Figma to design everything from scratch.

Instead, they use it to understand:

- Colors
- Fonts
- Font Sizes
- Margins
- Padding
- Images
- Icons
- Spacing
- Layout Structure

Then they recreate that design using **HTML & CSS**.

---

# рҹ–ҢпёҸ What is Penpot?

**Penpot** is another UI/UX design tool.

Unlike Figma, Penpot is **open source**.

It provides many similar features:

- UI Design
- Prototyping
- Collaboration
- Developer Handoff

Many developers prefer Penpot because it is free and open source.
